5. Database Setup (PostgreSQL Using Docker)

We run PostgreSQL using Docker Desktop.

Prerequisites:

Install Docker Desktop

Make sure Docker is running

PostgreSQL via Docker (example):

docker run -d \

--name bankstack-postgres \

-e POSTGRES_DB=postgres \

-e POSTGRES_USER=postgres \

-e POSTGRES_PASSWORD=postgres \

-p 5432:5432 \

-v bankstack_pgdata:/var/lib/postgresql/data \

postgres:15

This setup:

Uses a named volume (data persists across restarts)

Exposes PostgreSQL on port 5432

7. Messaging (Kafka and Kafka UI Using Docker)

Kafka is the event backbone of our system. We use it for:

Payment workflows

Sagas

Retries

Asynchronous processing

Kafka setup (Docker Compose example):

version: "3"

services:

zookeeper:

image: confluentinc/cp-zookeeper:7.6.0

environment:

ZOOKEEPER_CLIENT_PORT: 2181

kafka:

image: confluentinc/cp-kafka:7.6.0

depends_on:

- zookeeper

ports:

- "9092:9092"

environment:

KAFKA_BROKER_ID: 1

KAFKA_ZOOKEEPER_CONNECT: zookeeper:2181

KAFKA_ADVERTISED_LISTENERS: PLAINTEXT://localhost:9092

KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1

kafka-ui:

image: provectuslabs/kafka-ui

ports:

- "8079:8080"

environment:

KAFKA_CLUSTERS_0_NAME: local

KAFKA_CLUSTERS_0_BOOTSTRAPSERVERS: kafka:9092

Kafka UI:

Open: http://localhost:8079

View topics, messages, and consumer groups

We also have dedicated videos explaining Kafka concepts if you want deeper understanding.

8. Authentication and Identity (Auth0 Trial)

We use Auth0 for authentication and authorization.

Why Auth0:

Real-world identity provider

Secure JWT tokens

Supports machine-to-machine authentication

No custom authentication code

Auth0 account:

Sign up at https://auth0.com

Free tier is sufficient for this course

Auth0 Concepts Used in This Course

We use Auth0 in two ways:

User authentication

Users log in

Auth0 issues JWT tokens

Tokens secure REST APIs

Machine-to-machine (M2M)

No user involved

Services authenticate using client credentials

Used for Kafka-triggered flows

This is standard production architecture, not a workaround.

What You Need to Configure in Auth0
